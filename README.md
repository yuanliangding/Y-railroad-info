Thoughtworks的面试总结

第一面：HR面，或者技术面，这一面比较简单，简单了解个人情况，基本都会通过

第二面：笔试，java的会给三道算法题（三选一！！！），要求写个小程序（此程序涉及到较为严重的oo思想）（不能马虎，再写这个程序的时候可以看出你的代码的能力，是否完美，追求极致，并且在笔试后还会在第三面的时候把这个程序做一个运行，并且在程序上再加上一些小功能，因此程序需要具备可扩展性，“增加的新功能和体现扩展性”最好在最开始写的时候就加进来！！！）

第三面：HR和技术面，两个人应该是同时在场，刚开始或者是技术和你沟通，技术会让你把之前提交的笔试题拿出来做一次运行，看是否有bug，甚至是提重构、调优思路（这个可以提前准备）没有的话就会让你在现在的程序上增加新的需求（扩展的小程序在功能实现的基础上，最好要考虑到业务层面，和业务更贴合，命名，类的结构），这儿过了以后会是hr和技术一起面试（问拿着简历，根据简历来问问题，可以事先自己回顾一下，或者称之为——第四面）。

可能会聊到的问题（第一面）：

1.语言层面上的基础知识；

2.工作中遇到的什么问题，你是如果解决的，有什么难点；

3.根据简历上的来提问，比如项目经验，负责过什么样的项目，这个项目的周期是多长，有几个人一起负责，你在这个项目中担任的是什么角色，你的收获是什么等等；

4.平时是怎么学习的，是在网站上学习还是书本中学习，网站的话是什么网站（GitHub，Slack，CSDN等等），接下来会问你有木有在上面互动，有没有分享过自己的项目或者知识；这点主要是看你有没有奉献精神，和有没有分享精神；

5.技术面试的时候，主要是看你对于这个技术懂得了多少，到达什么程度，技术官的技术还是比较牛逼的，因此，不能不懂想蒙过去，这样会加分，不懂就直接说对这个了解不是很清楚或者直接说不清楚就可以了。

6.Thoughtworks是一家外资公司，英文好对相对的加分，所以提前准备好英文的自我介绍，以及英文的项目介绍是一个明确的选择，除此之外，千万一定不能用四川话来回答问题，至少是普通话；

如果你能再见一次HR，那么你的能力还有文化都是比较和thoughtworks很匹配的了，最后HR会和你聊聊你现在的薪资，期望的薪资，离职原因，上一个公司怎么样，未来三到五年的规划等等，在这儿要注意，态度要积极，尽量少或者不要吐槽上一家公司和领导，给hr留下一个比较好的意向，尽量用普通话或者英语交谈。
